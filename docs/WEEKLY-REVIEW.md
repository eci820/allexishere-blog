## 📌 복기 방법 — 숫자 어디서 가져오나

| 지표 | 출처 | 뽑는 법 |
|---|---|---|
| 생성 수 | 로컬 `automation/state/cost-log.jsonl` | 이번 주 생성 항목 줄 수 |
| 수정 횟수 | 같은 파일 | `"action":"edit"` 항목 수 |
| 승인(발행) 수 | git 로그 | `git log --since="7 days ago" --oneline \| grep -c "content: 발행"` |
| 반려 횟수 | 수기 | 텔레그램에서 [❌반려] 누른 횟수(기억/메모) |
| 노출수·클릭수·CTR·평균 게재순위 | **Google Search Console** → 실적(Performance) | 기간=이번 주, '페이지'별로 필터 |
| 색인 여부 | Search Console → 페이지 | 새 글이 '색인 생성됨'인지 |
| 광고 노출수·페이지뷰·수입 | **Google AdSense** → 보고서 | 기간=이번 주. 텍스트 우선 전환일(**2026-07-05**) 전후로 나눠 비교 |
| 구독 소진율 | Claude Code에서 **`/usage`** 입력 | 주간 사용량 ÷ 한도 |

> 빠른 집계(로컬): 아래를 터미널에 붙여넣으면 ①의 생성·수정·발행 수가 나옵니다.
> ```bash
> cd ~/Projects/allexishere-blog
> echo "생성: $(grep -c '"ts"' automation/state/cost-log.jsonl) (누적) / 수정: $(grep -c '\"action\":\"edit\"' automation/state/cost-log.jsonl)"
> git log --since="7 days ago" --oneline | grep -c "content: 발행" | sed 's/^/발행(승인) 수(최근 7일): /'
> ```
> ※ cost-log는 누적이라 '이번 주'만 세려면 `ts`의 날짜로 필터하세요(예: `grep 2026-07-1 automation/state/cost-log.jsonl | wc -l`).
