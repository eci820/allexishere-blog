# 블로그 웹 편집기 사용법 (Sveltia CMS)

브라우저에서 글을 쓰고 고칠 수 있는 편집기입니다.
**개발 모드에서만 켜지고**, 실제 배포되는 사이트(운영)에는 편집기가 들어가지 않습니다.

> ⚠️ **크롬(Chrome)·엣지(Edge)·브레이브 등 크로미움 계열 브라우저에서만** 동작합니다.
> (사파리·파이어폭스는 로컬 파일 편집 기능을 지원하지 않아요.)

---

## 1. 편집기 켜기

터미널에서 프로젝트 폴더로 가서 개발 서버를 켭니다.

```bash
cd ~/Projects/allexishere-blog
npm run dev
```

크롬 주소창에 아래를 입력해 편집기를 엽니다 (끝에 `index.html`까지 꼭 붙이세요):

```
http://localhost:4321/admin/index.html
```

처음 열면 **“Work with Local Repository”** 버튼이 보입니다.
1. 그 버튼을 누릅니다.
2. 폴더 선택 창이 뜨면 **이 프로젝트 폴더(`allexishere-blog`)를 통째로 선택**하고 “열기/허용”을 누릅니다.
   (한 번만 허용하면 됩니다.)
3. 왼쪽에 **“블로그 글”** 목록이 뜨고, 기존 글 92개가 그대로 보입니다.

---

## 2. 글 쓰기 / 고치기 / 저장

### 새 글 쓰기
1. 왼쪽 “블로그 글” → 오른쪽 위 **“New 글”(새 글)** 버튼.
2. 항목을 채웁니다:
   - **제목**: 글 제목
   - **발행일**: `2026-07-04T09:00:00+09:00` 형식으로 (끝의 `+09:00`까지 그대로 — 한국 시간 표시).
   - **카테고리**: `info` (그대로 두면 됩니다).
   - **본문**: 아래 “본문 편집” 참고.
   - 나머지(설명·태그·대표이미지 등)는 선택입니다.
3. 오른쪽 위 **“Save”**(저장)를 누르면 컴퓨터의 실제 `.md` 파일로 저장됩니다.

### 본문 편집 — 리치 에디터 ↔ 원문(마크다운) 토글
- 본문 편집기 오른쪽 위의 아이콘으로 **리치 텍스트 모드**와 **마크다운(원문) 모드**를 오갈 수 있습니다.
- **원문 그대로가 중요할 땐 마크다운 모드**로 쓰고 저장하세요(형식이 덜 바뀝니다).

### 사진 넣기
- 본문에서 이미지 버튼으로 사진을 추가하면 **그 글 폴더 안에 같이 저장**되고 `./파일명` 으로 연결됩니다(기존 구조와 동일).

### ⚠️ 기존 글 고칠 때 주의
- **원본 경로(originalPath)** 는 구글 검색 유입 보존용이니 **건드리지 마세요.**
- 기존 글을 편집기로 저장하면 본문 형식이 아주 약간 정리(재작성)될 수 있습니다.
  형식을 100% 그대로 두고 싶은 글은 편집기 대신 `.md` 파일을 직접 고치는 걸 권합니다.

---

## 3. 발행하기 (사이트에 반영)

편집기 저장은 **내 컴퓨터 파일에만** 반영됩니다. 실제 사이트에 올리려면 **발행**을 해야 합니다.

터미널에서 한 줄이면 끝납니다:

```bash
npm run publish
```

이 명령이 자동으로 **바뀐 파일 모으기 → 커밋 → GitHub 푸시**를 합니다.
푸시되면 **Cloudflare Pages가 1~3분 안에 자동으로 새로 배포**합니다.

> 바뀐 게 없으면 “올릴 것이 없어요”라고 알려주고 아무것도 하지 않습니다.

---

## 요약 흐름

```
npm run dev  →  localhost:4321/admin/index.html 에서 글 편집·저장  →  npm run publish
     (편집기 켜기)              (내 컴퓨터에 저장)                    (사이트에 발행)
```
