# 글쓰기 가이드

기본 글쓰기 도구는 **로컬 글쓰기 편집기**입니다. 티스토리처럼 "새 글쓰기 → 제목·본문
입력 → 발행 버튼" 흐름이라, **폴더·frontmatter·터미널 명령 없이** 글을 쓸 수 있습니다.

> 편집기는 **내 컴퓨터에서만**(개발 모드) 동작하고, 실제 배포 사이트(운영)에는 아무 영향이
> 없습니다. Toast UI Editor 3.2.2를 내장해 **인터넷 없이도** 열립니다.

---

## 1. 편집기 열기

터미널에서 한 줄:

```bash
cd ~/Projects/allexishere-blog
npm run write
```

→ 개발 서버가 켜지고 브라우저에 **글 목록 화면**이 자동으로 열립니다
(`http://localhost:4321/write`).

---

## 2. 글 목록 화면

- 전체 글이 보이고, 각 글에 **`발행` / `임시`** 배지가 붙습니다.
- **새로 쓴 글·임시글** = `열기`(편집 가능).
- **기존 92개 글** = `읽기전용` `보기`(v1에서는 안전을 위해 수정 잠금).
- 오른쪽 위 **[+ 새 글쓰기]** 로 새 글을 시작합니다.

---

## 3. 새 글쓰기

1. **[+ 새 글쓰기]** 클릭.
2. **제목** 입력 → 아래 **주소(슬러그)** 가 자동 생성됩니다.
   - 주소는 **글 웹주소**가 됩니다: `allexishere.com/entry/주소`.
   - **발행 전까지** 직접 수정 가능. **발행하면 주소가 고정**되어, 이후 제목을 바꿔도
     주소는 변하지 않습니다(검색 유입 보존).
3. **태그**(Enter로 추가)·**설명**(요약) 입력.
4. **본문** 작성 — 위지윅(보이는 대로)과 마크다운을 자유롭게 전환. 표·목록·링크·이미지
   툴바 제공.
5. 커스텀 버튼:
   - **🖼 갤러리(여러 장)**: 사진 여러 장 선택 → 글 폴더에 저장(파일명 자동 안전화) →
     **연속 이미지**로 삽입(사이트에서 자동 3열 그리드).
   - **💡 팁박스**: 팁/주의/참고 선택 → 강조박스 삽입.
6. 아래 버튼:
   - **[임시저장]**: 글을 저장(아직 사이트엔 안 나감, `임시` 상태).
   - **[미리보기]**: 저장한 글을 `localhost`에서 **새 탭으로** 실제 모습 확인.
   - **[발행하기]**: 사이트에 올립니다(자동으로 **그 글만** 커밋+푸시). **이 버튼이
     유일한 발행 방법** — 다른 저장은 사이트에 영향 없습니다. 발행하면 Cloudflare가
     1~3분 뒤 자동 배포합니다.

> 사진은 **base64로 박히지 않고** 항상 글 폴더에 파일로 저장됩니다(`./파일명`).

---

## 4. 요약 흐름

```
npm run write → [새 글쓰기] → 제목·본문·사진·팁박스 → [임시저장] → [미리보기] → [발행하기]
```

---

## (선택) 다른 도구들

기본은 위 편집기지만, 원하면 아래도 쓸 수 있습니다(둘 다 개발 모드 전용).

- **Obsidian**: `src/content/blog` 폴더를 보관함으로 열어 `.md`를 직접 편집. 마크다운을
  손으로 다루는 데 익숙하면 편합니다. (새 글은 폴더 만들고 `index.md` 작성 — 폴더명=주소)
- **Sveltia CMS** (`http://localhost:4321/admin/index.html`): 전체 글 목록·frontmatter를
  한눈에 확인하는 보조용.

> 기존 92개 글을 고쳐야 할 때는, 위지윅 재저장으로 인한 본문 훼손을 피하려면 당분간
> Obsidian이나 `.md` 직접 편집을 쓰세요(편집기의 기존 글 수정 개방은 diff 가드와 함께 v2 예정).

## 📂 기존 글 배지 규칙 (트렌드 파이프라인 연계)

브리핑·초안 카드에 **📂(기존 글 보유)** 배지가 뜨면, 같은 주제의 발행 글이 이미 있다는 신호입니다.

- **탭해서 새로 만들지 마세요.** 새 초안은 기존 글과 주제가 겹쳐 자기잠식(중복)이 됩니다.
- 대신 **기존 글을 "갱신"하도록 지시**하세요(이 편집기 `npm run write` 에서 해당 글 열어 보강,
  또는 봇에게 그 글 기준으로 업데이트 지시).
- 배지는 **자동 갱신을 하지 않습니다** — 사람이 판단해 "갱신 지시"를 하라는 신호일 뿐입니다.
- `/draft <주제>` 도 기존 글과 겹치면 경고 후 [계속(새 초안)]/[취소] 를 물어봅니다.

## ✏️ 텔레그램 초안 수정 (파이프라인, v2.4)

봇이 보낸 초안 카드에서 **[✏️수정]**을 누르면 발행 전 초안을 고칠 수 있습니다(대원칙 불변: **발행은 사람 승인만**, 수정은 발행 전에만).

- **[✏️수정] 탭 → 안내 메시지에 "답장(reply)"으로 지시**:
  - `제목: 새로운 제목` → 제목·주소(슬러그) 교체(발행 전이므로 주소 변경 OK).
  - `첫 문단 쉽게` / `표에 9월분 행 추가` / `끝에 주의 한 줄` → **claude-cli(구독)로 지정 부분만 수정**(나머지 본문·표·목록 구조 유지).
- 수정 후 **변경된 초안 카드가 다시 옵니다** → [전문]으로 확인, 여러 번 수정 가능, [✅승인]으로 게시.
- **안전장치**: draft(발행 전)만 수정 가능. 발행된 글은 차단(편집기 또는 별도 지시로). AI 수정 시 원본은 `automation/state/backups/`에 백업되고, 결과가 지시 범위를 벗어나 길이가 급변하면 경고가 표시됩니다.
- **표·이미지 정밀 편집은 이 편집기(`npm run write`)가 더 정확합니다** — 표 셀/이미지 배치처럼 세밀한 작업은 편집기에서 하세요.
