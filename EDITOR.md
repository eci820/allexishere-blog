# 글쓰기 가이드 (Obsidian)

이제 **Obsidian**(옵시디언)으로 글을 씁니다. 마크다운 `.md` 파일을 그대로 편집하는
무료 앱이라, 기존 글 형식을 바꾸지 않고 안전하게 쓸 수 있습니다.

> 편집기는 **내 컴퓨터에서만** 동작하고, 실제 배포되는 사이트(운영)에는 아무 영향이 없습니다.

---

## 1. Obsidian 설치 & 보관함 열기 (처음 한 번)

1. https://obsidian.md 에서 앱을 내려받아 설치합니다.
2. Obsidian 실행 → **"Open folder as vault"(폴더를 보관함으로 열기)** →
   **`allexishere-blog/src/content/blog`** 폴더를 선택합니다.
3. 왼쪽에 기존 글 폴더(1, 2, 3 … 99)가 쭉 보이면 성공입니다.

> 링크·첨부파일·템플릿 설정은 이미 맞춰 두었습니다(위키링크 끔 / 첨부는 같은 폴더 /
> 링크 자동 업데이트 / 템플릿 폴더 `_templates`). 그냥 쓰시면 됩니다.

---

## 2. 새 글 쓰는 규칙 (중요)

### ✅ 폴더 이름 = 글 주소(슬러그)
새 글은 **폴더를 하나 만들고 그 안에 `index.md`** 를 두는 구조입니다.
**폴더 이름이 그대로 웹 주소가 됩니다.**

| 만든 폴더 | 글 주소 |
|---|---|
| `여름-휴가-준비물` | `allexishere.com/entry/여름-휴가-준비물` |
| `2026-연말정산-팁` | `allexishere.com/entry/2026-연말정산-팁` |

규칙:
- 폴더 이름은 **띄어쓰기 대신 하이픈(-)**, 한글/영문/숫자 OK, **슬래시(/)·특수문자 금지**.
- 폴더 안 파일은 **반드시 `index.md`** 여야 합니다(다른 이름이면 글로 안 잡힘).
- 한 번 정한 폴더 이름(주소)은 **나중에 바꾸지 마세요**(검색 유입이 끊깁니다).

> **기존 글(숫자 폴더 1~99)은 이름을 바꾸지 마세요.** 그 글들의 주소는 `originalPath`
> 값으로 보존되고 있어서, 폴더명을 건드리면 안 됩니다. 내용 수정만 하세요.

### ✅ 새 글 만드는 순서
1. 보관함에서 **새 폴더** 생성 (이름 = 원하는 주소).
2. 그 안에 **`index.md`** 새 노트 생성.
3. 명령 팔레트(`Cmd/Ctrl + P`) → **"Templates: Insert template"** → **`새-글`** 선택
   → 기본 틀(제목·요약·날짜·태그·`draft: true`)이 들어옵니다.
4. 제목·본문을 채웁니다. **날짜는 `2026-07-04T09:00:00+09:00` 형식**(끝의 `+09:00`까지).
5. 쓰는 동안은 `draft: true` 로 두세요(사이트에 안 나옴). **발행 준비되면 그 줄을 지우거나
   `draft: false`** 로 바꿉니다.

### ✅ 사진 넣기
- 이미지 파일을 **그 글 폴더 안에** 넣고, 본문에서 `![](./파일명.png)` 로 씁니다.
- **사진 2장 이상을 줄줄이** 넣으면 자동으로 **격자(그리드)** 로 묶여 보입니다
  (데스크톱 3열·모바일 2열). 한 장만 넣으면 크게 보입니다.

### ✅ 강조박스(콜아웃) 3종
```
> [!tip] 팁 제목
> 강조할 팁 내용.

> [!warning]
> 주의할 점. (제목 생략하면 '주의'로 표시)

> [!note] 참고 제목
> 참고 내용.
```
→ 사이트에서 팁/주의/참고 색깔 박스로 예쁘게 나옵니다.

## 3. 미리보기 → 발행

### 미리보기 (내 컴퓨터에서)
```bash
cd ~/Projects/allexishere-blog
npm run dev
```
크롬에서 `http://localhost:4321` 접속. **개발 미리보기에서는 `draft: true` 글도 보입니다**
(발행 전에 확인용). 주소로 바로 보려면 `http://localhost:4321/entry/폴더이름`.

### 발행 (사이트에 반영)
Obsidian 저장은 내 컴퓨터 파일에만 반영됩니다. 실제 사이트에 올리려면:
```bash
npm run publish
```
→ 바뀐 파일을 모아 커밋하고 GitHub에 올립니다. **Cloudflare가 1~3분 뒤 자동 배포**합니다.
(발행하면 `draft: true` 인 글은 여전히 사이트에 안 나옵니다 — 공개하려면 draft를 false로.)

---

## 4. 요약 흐름

```
Obsidian에서 새 폴더+index.md 작성  →  npm run dev 로 미리보기  →  draft 해제  →  npm run publish
```

## 보조 도구: Sveltia CMS (/admin)

브라우저 편집기 **Sveltia**도 그대로 남겨뒀습니다. 글쓰기 주력은 Obsidian이지만,
**전체 글 목록을 한눈에 보거나 frontmatter 값(날짜·태그·원본경로 등)을 일괄 확인**할 때
보조로 쓰기 좋습니다.

- 여는 법: `npm run dev` 상태에서 크롬 → `http://localhost:4321/admin/index.html`
  → "Work with Local Repository" → 프로젝트 폴더 허용.
- 이것도 **개발 모드 전용**이라 운영 사이트에는 포함되지 않습니다.
